# EduAI Copilot

面向大学生的 AI 学习助手产品调研与 MVP 原型

## 项目简介

这是为 AI 产品实习生岗位准备的作品集项目。做了一个完整的 HTML 报告，展示了从用户调研到产品设计的全过程。

项目定位是"学习伙伴"而不是"答案机器"，核心思路是引导学生思考，而不是直接给答案。

## 项目背景

大学生学习时经常遇到这些问题：课上老师讲太快跟不上，课后笔记零散不知道怎么整理，考前复习没计划效率低。

现有的 AI 工具比如 ChatGPT 能回答问题，但没有针对学习场景做优化，给的答案太直接，学生看完还是不会。

所以想做一个专门给大学生用的 AI 学习助手，覆盖从笔记整理到复习计划的完整学习流程。

## 目标用户

主要面向三类大学生：

**考研党**
- 大三为主，理工科居多
- 痛点：复习资料多、概念理解不深、时间紧
- 需求：高效整理笔记、深入理解概念、制定复习计划

**就业党**
- 大三为主，计算机/软件工程居多
- 痛点：缺乏项目经验、面试准备不充分
- 需求：整理技术知识、模拟面试问题

**普通大学生**
- 大二大三，文理科都有
- 痛点：课后复习效率低、作业不会做
- 需求：整理课堂笔记、解释难懂概念

## 核心功能

### P0 核心功能（必须有）

| 功能 | 说明 |
|------|------|
| 课程资料总结 | 上传课件或笔记，AI 生成章节摘要、知识点列表和考试重点 |
| 概念解释助手 | 针对难懂概念，用定义、类比、例题、误区四种方式解释 |
| 复习计划生成 | 根据考试时间和内容，生成每日复习计划 |

### P1 重要功能（最好有）

| 功能 | 说明 |
|------|------|
| 练习题生成 | 根据知识点生成选择题、简答题和解析 |
| 学习反馈分析 | 分析错题原因，给出学习建议 |

### P2 扩展功能（以后做）

| 功能 | 说明 |
|------|------|
| 课程知识库 | 构建课程知识库，支持更精准的问答 |
| 学习进度追踪 | 记录学习进度，分析学习效果 |

## 项目亮点

1. **做了完整的用户调研**：访谈了 10 个大学生，发了 100 份问卷，痛点有数据支撑

2. **竞品分析有深度**：对比了 ChatGPT、Kimi、豆包、通义千问、Notion AI 五个产品，找了 8 个维度打分

3. **Prompt 设计有方法**：为 5 个核心场景设计了专门的 Prompt 模板，每个都有输入输出示例

4. **原型看起来像真的**：做了 4 个静态原型界面，样式像真实产品

5. **数据可视化**：用 Chart.js 做了 6 个图表，展示调研数据和竞品对比

## 技术栈

- HTML5 + CSS3 + 原生 JavaScript
- Chart.js 做图表
- 没用框架，纯静态页面
- 部署在 GitHub Pages 上

## 本地运行

```bash
# 克隆仓库
git clone https://github.com/yourusername/eduai-copilot-portfolio.git
cd eduai-copilot-portfolio

# 启动本地服务器
python -m http.server 8000

# 访问 http://localhost:8000
```

也可以直接双击 `index.html` 打开，但图表需要网络连接才能加载 Chart.js CDN。

## GitHub Pages 部署

1. 推送代码到 GitHub
2. 在仓库设置中启用 GitHub Pages
3. 选择 `main` 分支作为发布源
4. 访问 `https://yourusername.github.io/eduai-copilot-portfolio/`

## 数据说明

项目中的数据都是模拟的，但参考了真实的调研方法：

- 用户调研：模拟了 30 名大学生的问卷数据
- 竞品评分：基于实际使用体验打分
- 痛点数据：参考了知乎、小红书上大学生的真实反馈
- 功能优先级：基于用户价值和实现难度综合判断
