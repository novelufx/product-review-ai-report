项目背景与目标

为什么做这个项目

大学生学习时经常遇到这些问题：课上老师讲太快跟不上，课后笔记零散不知道怎么整理，考前复习没计划效率低。

现有的 AI 工具比如 ChatGPT 能回答问题，但没有针对学习场景做优化，给的答案太直接，学生看完还是不会。

产品定位

我想做一个专门给大学生用的 AI 学习助手，叫 EduAI Copilot。

它的定位是"学习伙伴"而不是"答案机器"，核心是引导学生思考，而不是直接给答案。

项目目标

  • 验证"学习伙伴"定位是否成立
  • 跑通 5 个核心学习场景
  • 设计 20+ 个 Prompt 模板
  • 做个可交互的 MVP 原型

大三为主，计算机/软件工程居多

需求：

核心使用场景

01

课后笔记整理

学生上完课，把零散的笔记丢给 AI，AI 帮他整理成结构化的知识总结

02

概念深度解释

学生遇到不懂的概念，AI 用通俗类比、技术定义、实际案例三种方式解释

03

学习计划生成

学生要考试了，AI 根据考试时间和内容安排每天的复习计划

04

错题分析

学生做错题了，AI 分析错因、讲解正确解法、再给几道类似的题巩固

05

知识点梳理

学生想系统复习，AI 帮他梳理知识点之间的关系，给出学习路径

竞品共同弱点

1

学习场景优化不足

都是通用工具，没针对学习场景做专门设计

2

产品机会点

🎯

专注学习场景

所有竞品都是通用工具，没有专门针对学习场景优化的。我们可以专门设计学习相关的功能。

💡

引导思考而非给答案

竞品大多直接给答案，学生看完还是不会。我们可以设计引导式提问，培养学生的思考能力。

🔗

覆盖完整学习流程

💰

价格优势

竞品要么贵，要么功能受限，我们可以基础功能免费，付费版价格亲民。

课程资料总结

P0

用户上传课件、笔记或文档后，AI 自动生成章节摘要、知识点列表和考试重点

用户场景

学生上完课，把零散的笔记、PPT 拍照或文档丢给 AI，AI 帮他整理成结构化的知识总结，省去手动整理的时间。

核心价值

  • 节省整理笔记的时间
  • 自动提取考试重点
  • 生成结构化知识总结

AI 能力

长文本总结 信息抽取 结构化输出
💡

概念解释助手

P0

针对难懂概念，使用定义、类比、例题、常见误区的方式进行解释

用户场景

学生遇到不懂的概念（比如"过拟合"、"导数"），问 AI，AI 用通俗类比、技术定义、实际案例、常见误区四种方式解释，让学生真正理解。

核心价值

  • 降低理解门槛
  • 多角度解释同一概念
  • 避免常见误区

AI 能力

Prompt Engineering 知识解释 多轮对话
📅

复习计划生成

P0

根据考试时间、章节范围、学生薄弱点和每日可用时间生成复习计划

用户场景

学生要考试了，告诉 AI 考试时间、要复习的内容、每天能学多久，AI 帮他安排每天的复习计划，标注重点和难点。

核心价值

  • 合理分配复习时间
  • 明确每天的学习目标
  • 根据薄弱点个性化推荐

AI 能力

任务拆解 计划生成 个性化推荐
✏️

练习题生成

P1

根据知识点生成选择题、简答题和解析

用户场景

学生想巩固某个知识点，告诉 AI 知识点和难度，AI 生成几道练习题，包含选择题、简答题，每道题都有详细解析。

核心价值

  • 针对性巩固练习
  • 题目类型多样
  • 详细解析帮助理解

AI 能力

AIGC 内容生成 题目生成 难度控制
📊

学习反馈分析

P1

根据学生答题结果分析薄弱知识点，给出下一步学习建议

用户场景

学生做完题或考完试，把错题给 AI，AI 分析每道题的错因，找出知识薄弱点，给出针对性的学习建议和巩固练习。

核心价值

  • 精准定位薄弱环节
  • 个性化学习建议
  • 推荐巩固练习

AI 能力

数据分析 反馈生成 学习路径优化

Prompt 模板

📄

课程资料总结 Prompt

使用场景

用户上传课件、笔记或文档后，系统调用此 Prompt 生成章节摘要、知识点列表和考试重点。

输入变量
{course_name} {chapter_title} {raw_content}
预期输出格式

Markdown 格式，包含：章节摘要、核心知识点列表、考试重点标注、学习建议

Prompt 
你是一位专业的课程学习助手。请根据以下课程资料生成结构化的学习总结。

课程名称：{course_name}
章节标题：{chapter_title}

课程资料：
{raw_content}

请按以下格式输出：

## 章节摘要
用 3-5 句话概括本章节的核心内容。

## 核心知识点
- 知识点 1：简要说明
- 知识点 2：简要说明
- ...

## 考试重点
标注可能作为考试重点的内容，使用 ⭐ 标记。

## 学习建议
给出 2-3 条针对性的学习建议。
💡

概念解释 Prompt

使用场景

用户提出难懂概念时，系统调用此 Prompt 从多个角度进行解释，帮助用户真正理解。

输入变量
{concept_name} {course_context}
预期输出格式

Markdown 格式，包含：通俗类比、技术定义、实际案例、常见误区

Prompt 
你是一位善于解释复杂概念的学习导师。请用多种方式解释以下概念，帮助学生真正理解。

概念名称：{concept_name}
课程背景：{course_context}

请按以下格式输出：

## 通俗类比
用日常生活中的例子来解释这个概念，让没有专业背景的人也能理解。

## 技术定义
给出准确的技术定义，包含关键术语的解释。

## 实际案例
举一个具体的应用案例，展示这个概念在实际中是如何使用的。

## 常见误区
列出学生在理解这个概念时常见的错误认识，并给出正确理解。
📅

复习计划生成 Prompt

使用场景

用户需要制定复习计划时，系统调用此 Prompt 根据考试时间和内容生成每日复习安排。

输入变量
{exam_date} {chapters} {daily_hours} {weak_points}
预期输出格式

Markdown 格式，包含：每日复习计划、重点标注、学习建议

Prompt 
你是一位学习规划师。请根据以下信息制定详细的复习计划。

考试日期：{exam_date}
需要复习的章节：{chapters}
每天可用学习时间：{daily_hours} 小时
学生的薄弱点：{weak_points}

请按以下格式输出：

## 复习计划

### Day 1：章节名称（X 小时）
- 上午：复习内容（时长）
  - 重点：标注重点内容
- 下午：复习内容（时长）
  - 重点：标注重点内容

### Day 2：...

## 学习建议
- 给出 3-5 条针对性的复习建议
- 针对薄弱点给出专项复习建议
✏️

练习题生成 Prompt

使用场景

用户需要巩固某个知识点时，系统调用此 Prompt 生成针对性的练习题。

输入变量
{knowledge_point} {difficulty} {question_count} {question_types}
预期输出格式

Markdown 格式，包含：题目、选项、答案、详细解析

Prompt 
你是一位经验丰富的出题老师。请根据以下要求生成练习题。

知识点：{knowledge_point}
难度等级：{difficulty}
题目数量：{question_count}
题目类型：{question_types}

请按以下格式输出每道题：

## 题目 X（题型）

题目内容

A. 选项 A
B. 选项 B
C. 选项 C
D. 选项 D

**答案：X**

**解析：**
详细解释为什么选这个答案，涉及哪些知识点，以及常见的错误理解。
📊

学习反馈分析 Prompt

使用场景

用户完成练习或考试后，系统调用此 Prompt 分析错题原因，给出学习建议。

输入变量
{wrong_questions} {user_answers} {correct_answers}
预期输出格式

Markdown 格式，包含：错因分析、薄弱知识点、学习建议、巩固练习推荐

Prompt 
你是一位学习分析师。请分析以下错题，找出学生的薄弱环节并给出改进建议。

错题信息：
{wrong_questions}

学生答案：{user_answers}
正确答案：{correct_answers}

请按以下格式输出：

## 错题分析

### 题目 X
**错因分析：**分析学生为什么做错（概念不清、计算错误、审题不仔细等）

**涉及知识点：**列出这道题考察的知识点

## 薄弱知识点
总结学生在哪些知识点上存在不足。

## 学习建议
针对薄弱知识点给出具体的学习建议。

## 巩固练习推荐
推荐 3-5 道类似的练习题，帮助学生巩固。

最终总结

主要发现

  • 大学生学习痛点集中在"记不住"、"来不及"、"听不懂"
  • 现有 AI 工具没有针对学习场景优化
  • 学生想要"学习伙伴"，不是"答案机器"
  • 从笔记到复习的完整流程，才是 AI 学习助手该覆盖的

产品价值

  • 专注学习场景，做深度优化
  • 引导思考而非直接给答案
  • 覆盖完整学习流程
  • 个性化学习路径

个人收获

  • 走通了从用户调研到产品设计的完整流程
  • 知道怎么设计好用的 Prompt 了
  • 理解了 AI 产品和传统产品有什么不同
  • 产品文档和可视化的水平提高了

项目反思

🔄 如果重新做，会改进什么

  • 做真实的用户访谈，而不是模拟数据
  • 做一个可交互的原型，而不只是静态页面
  • 接入真实的 AI API，展示实际效果
  • 做 A/B 测试，验证 Prompt 效果

💡 对 AI 产品的新理解

  • AI 产品不是"有 AI 就行"，关键是要解决真实问题
  • Prompt 设计是 AI 产品的核心竞争力
  • 用户不关心你用了什么模型，只关心能不能帮到他
  • AI 产品的迭代速度比传统产品快很多

🎯 面试时会怎么讲这个项目

  • 先讲用户痛点，再讲产品方案
  • 重点讲 Prompt 设计的思路和迭代过程
  • 展示数据分析能力，用数据支撑产品决策
  • 承认不足，说明改进方向

核心结论

EduAI Copilot 的定位是"学习伙伴"而不是"答案机器"。通过专注学习场景、引导思考、覆盖完整学习流程，可以在竞品中找到差异化的机会。
