# 用户研究文档

## 1. 调研目标

搞清楚大学生学习时遇到什么问题，他们需要什么样的 AI 工具，为产品设计提供依据。

具体想了解：
- 大学生学习时最大的痛点是什么
- 他们现在用什么工具学习，效果怎么样
- 他们希望 AI 学习助手帮他们解决什么问题
- 他们愿意为这样的工具付多少钱

## 2. 问卷设计

### 基本信息
1. 你目前大几？
   - 大一 / 大二 / 大三 / 大四 / 研究生

2. 你是什么专业？
   - 理工科 / 文科 / 商科 / 艺术 / 其他

3. 你目前的学习目标是什么？
   - 考研 / 就业 / 出国 / 其他

### 学习痛点
4. 你学习时遇到的最大困难是什么？（多选）

   - 课后记不住
   - 作业不会做
   - 考前来不及

   - 其他

   - 手写笔记
   - 拍照/PPT

   - 内容太多
   - 没有计划
   - 理解不深

### 工具使用

   - 用过 ChatGPT
   - 用过 Kimi
   - 用过豆包
   - 用过其他
   - 没用过

8. 你觉得这些 AI 工具学习效果怎么样？

   - 一般，有时候有用
   - 不好，答案太直接
   - 没用过

   - 直接给我答案
   - 引导我思考

   - 帮我制定计划

### 付费意愿
10. 你愿意为 AI 学习工具付多少钱？
    - 免费
    - 10-30 元/月
    - 30-50 元/月
    - 50 元以上/月

## 5. 需求分析

### 核心需求

| 需求 | 描述 | 优先级 | 用户占比 |
|------|------|--------|----------|
| 笔记整理 | 帮我把零散笔记整理好 | P0 | 85% |
| 概念解释 | 用我能懂的方式解释概念 | P0 | 80% |
| 学习计划 | 帮我安排复习计划 | P0 | 75% |
| 错题分析 | 告诉我为什么做错 | P1 | 65% |
| 知识梳理 | 帮我理清知识关系 | P1 | 55% |

### 需求优先级

```
紧急程度 ↑
         |
    P0   |  笔记整理  概念解释  学习计划
         |
    P1   |  错题分析  知识梳理
         |
         +------------------------→ 重要程度
```

### 用户期望

**学习伙伴，不是答案机器**
> "我想要的是能引导我思考的工具，不是直接给我答案。"
> —— 理科生小刘

**简单好用**
> "界面要简单，操作不要太复杂。"
> —— 文科生小张

**个性化**
> "每个人学习情况不一样，工具要能根据我的情况给建议。"
> —— 考研党小王

## 6. 调研结论

### 主要发现

1. **痛点集中**：大学生学习痛点主要集中在"记不住"、"来不及"、"听不懂"三个方面

2. **工具不足**：现有 AI 工具没有针对学习场景优化，答案太直接，不能引导思考

3. **需求明确**：学生最需要的是笔记整理、概念解释、学习计划三个功能

4. **定位清晰**：学生想要"学习伙伴"，不是"答案机器"

5. **付费意愿**：大部分学生愿意为好用的学习工具付费，但价格敏感

### 产品方向

- 聚焦 P0 核心功能，先跑通三个场景
- 定位"学习伙伴"，引导思考而非直接给答案
- 界面简单，操作方便
- 价格亲民，学生能接受

### 下一步

- 根据调研结果设计产品功能
- 设计 Prompt 模板
- 做 MVP 原型测试
