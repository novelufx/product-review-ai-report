# EduAI Copilot 产品需求文档

## 1. 产品背景

大学生学习时经常遇到这些问题：课上老师讲太快跟不上，课后笔记零散不知道怎么整理，考前复习没计划效率低。现有的 AI 工具比如 ChatGPT 能回答问题，但没有针对学习场景做优化，给的答案太直接，学生看完还是不会。

我想做一个专门给大学生用的 AI 学习助手，叫 EduAI Copilot。它的定位是"学习伙伴"而不是"答案机器"，核心是引导学生思考，而不是直接给答案。

## 2. 目标用户

主要面向这几类大学生：

**考研党**
- 大三为主，理工科居多
- 痛点：复习资料多、概念理解不深、时间紧
- 需求：高效整理笔记、深入理解概念、制定复习计划

**就业党**
- 大三为主，计算机/软件工程居多
- 痛点：缺乏项目经验、面试准备不充分
- 需求：整理技术知识、模拟面试问题

**普通大学生**
- 大二大三，文理科都有
- 痛点：课后复习效率低、作业不会做
- 需求：整理课堂笔记、解释难懂概念

## 3. 核心场景

### 场景 1：课后笔记整理
学生上完课，把零散的笔记丢给 AI，AI 帮他整理成结构化的知识总结，包含核心概念、关键公式、实际应用。

### 场景 2：概念深度解释
学生遇到不懂的概念，问 AI，AI 用通俗类比、技术定义、实际案例三种方式解释，让学生真正理解。

### 场景 3：学习计划生成
学生要考试了，告诉 AI 考试时间、要复习的内容、每天能学多久，AI 帮他安排每天的复习计划。

### 场景 4：错题分析
学生做错题了，把题目给 AI，AI 分析错因、讲解正确解法、再给几道类似的题巩固。

### 场景 5：知识点梳理
学生想系统复习某个主题，AI 帮他梳理知识点之间的关系，画出知识图谱，给出学习路径。

## 4. 用户痛点

根据调研（模拟），大学生学习主要有这些痛点：

| 痛点 | 描述 | 影响程度 |
|------|------|----------|
| 课上听不懂 | 老师讲太快，跟不上节奏 | 高 |
| 课后记不住 | 笔记零散，不知道重点 | 高 |
| 作业不会做 | 缺乏思路，没人指导 | 中 |
| 考前来不及 | 复习没计划，效率低 | 高 |
| 理论用不上 | 学的东西不知道怎么用 | 中 |

## 5. 产品目标

### 短期目标（MVP）
- 验证"学习伙伴"定位是否成立
- 跑通笔记整理、概念解释、学习计划三个核心场景
- 收集 10 个用户的反馈

### 中期目标
- 覆盖 5 个核心学习场景
- 用户满意度达到 4 分以上（5 分制）
- 日活跃用户达到 100 人

### 长期目标
- 成为大学生首选的 AI 学习工具
- 建立学习社区
- 实现商业化

## 6. MVP 功能列表

### P0 核心功能（必须有）

| 功能 | 描述 | 输入 | 输出 |
|------|------|------|------|
| 笔记整理 | 把零散笔记整理成结构化知识 | 课堂笔记文本 | 结构化知识总结 |
| 概念解释 | 用多种方式解释难懂的概念 | 概念名称或问题 | 通俗类比+技术定义+案例 |
| 学习计划 | 根据情况生成复习计划 | 考试时间、内容、可用时间 | 每日学习计划表 |

### P1 重要功能（最好有）

| 功能 | 描述 | 输入 | 输出 |
|------|------|------|------|
| 错题分析 | 分析错因，给相似题巩固 | 错题内容 | 错因分析+相似题 |
| 知识图谱 | 梳理知识点关系 | 知识点名称 | 知识图谱+学习路径 |

### P2 扩展功能（以后做）

| 功能 | 描述 |
|------|------|
| 学习社区 | 和同学分享学习心得 |
| 进度追踪 | 记录学习进度 |
| 个性化推荐 | 根据学习情况推荐内容 |

## 7. 功能优先级

```
优先级矩阵

紧急程度 ↑
         |
    P0   |  笔记整理  概念解释  学习计划
         |
    P1   |  错题分析  知识图谱
         |
    P2   |  学习社区  进度追踪
         |
         +------------------------→ 重要程度
```

P0 功能优先实现，确保核心场景跑通。P1 功能在 MVP 验证后做。P2 功能等用户量起来再说。

## 8. 用户流程

### 笔记整理流程
```
学生输入笔记 → AI 分析内容 → 生成结构化总结 → 学生查看/修改 → 保存
```

### 概念解释流程
```
学生提问 → AI 识别概念 → 生成三种解释 → 学生追问/确认 → 理解完成
```

### 学习计划流程
```
学生输入考试信息 → AI 分析内容量 → 生成每日计划 → 学生调整 → 执行计划
```

### 错题分析流程
```
学生输入错题 → AI 分析错因 → 讲解正确解法 → 给相似题 → 学生练习
```

## 9. 验收标准

### 笔记整理
- 能识别课堂笔记中的核心概念
- 输出格式规范，包含概念、公式、应用
- 响应时间 < 3 秒

### 概念解释
- 能用通俗语言解释专业概念
- 包含类比、定义、案例三种形式
- 学生能看懂（测试：找 3 个学生试用）

### 学习计划
- 计划合理，每天的任务量适中
- 包含具体内容和时长
- 学生觉得能做到（测试：找 3 个学生试用）

### 错题分析
- 能找出做错的根本原因
- 讲解清晰，学生能理解
- 相似题质量高，确实能巩固

## 10. 后续迭代计划

### 第一阶段：MVP（1-2 周）
- 实现 P0 核心功能
- 做个简单的网页原型
- 找 10 个学生试用，收集反馈

### 第二阶段：优化（2-4 周）
- 根据反馈优化 Prompt
- 加上 P1 功能（错题分析、知识图谱）
- 优化界面和交互

### 第三阶段：扩展（1-2 月）
- 加上 P2 功能
- 做移动端适配
- 考虑商业化

### 第四阶段：推广（2-3 月）
- 在校园推广
- 建立用户社区
- 持续迭代优化
